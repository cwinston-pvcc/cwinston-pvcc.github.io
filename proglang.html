Programming Algorithms & Software Testing Terminology Table
TERM DEFINITION
machine language programs consisting of sequences of instructions that are encoded as numeric digits
debugging locating and correcting errors in a program
assembler programs which convert mnemonic expressions into machine language
assembly language a mnemonic system for representing programs
translator a program which translates programs expressed in high-level primitives into machine-language programs
compiler a program which translates several machine instructions into short sequences to simulate the activity requested by a single high-level primitive
interpreter programs which execute instructions as they are translated instead of recording the translated version for future use
cross-platform software software that is independent of an operating system's design as well as the machine's hardware design and is therefore executable throughout a network
natural language languages such as English, German, and Latin which evolved over time without formal grammatical analysis
formal language languages such as programming languages that are precisely defined by grammars
machine independence programs written in a third-generation language which does not refer to the attributes of any particular machine
high-level programming language a programming language with strong abstraction from the details of the computer
object-oriented programming (OOP) software in which the system is viewed as a collection of units, called objects, each of which is capable of performing the actions that are immediately related to itself as well as requesting actions of other objects
scripting language a programming language that employs a high-level construct to interpret and execute one command at a time. This language is typically used to perform administrative tasks rather than to develop complex programs.
Data types:
  • integer
    		•  numeric data consisting of whole numbers, probably stored using two's complement notation
  • float
    		•  numeric data that might contain values other than whole numbers, probably stored in floating-point notation
  • string
    		•  a sequence of characters, either as a literal constant or as some kind of variable
    primitive data type the data types that are included as promitives in a programming language, such as int for integer and char for character
    data structure the conceptual shape or attangement of data
    Python collection data structures
  • list
    		•  Lists are used to store multiple items in a single variable. List items are ordered, changeable, and allow duplicate values.
  • tuple
    		•  Tuples are also used to store multiple items in a single variable. Tuple items are ordered, unchangeable, and allow duplicate values
  • set
    		•  Sets are used to store multiple items in a single variable. Set items are unordered, unchangeable, and do not allow duplicate values.
  • dictionary
    		•  Dictionaries are used to store data values in key: value pairs. A dictionary is a collection which is ordered, changeable and do not allow duplicates
    function a set of instructions for performing a task that can be used as an abstract tool by other program units
    local variable a variable declared within a function
    global variable a variable whose scope is not restricted to a particular part of a program
    source program a program written in a language other than machine code, typically a high-level language
    object program a program into which some other program is translated by an assembler or compiler
